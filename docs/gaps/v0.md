  Gaps before implementation

  1. No LLM provider strategy for reconciliation/consolidation. The architecture requires LLM calls for reconciliation (ADD/UPDATE/DELETE/NOOP), consolidation (episode → concept/procedure/reflection), and
  importance scoring — but there's no ADR or design for how Cerebro talks to an LLM. This is different from embeddings. We need a similar pluggable provider story for the "thinking" model.

  2. No prompt engineering specification. The reconciliation and consolidation operations depend heavily on prompt design. What does the reconciliation prompt look like? What instructions does the consolidation LLM
   receive? These are critical for correctness.

  3. No error handling / resilience design. What happens when the LLM is unavailable during reconciliation? Do we fall back to always-ADD? What about partial failures during consolidation?

  4. No CLI command specification. We have remember, recall, reflect, forget, promote, consolidate, gc, inspect, export, init mentioned — but no formal command spec (flags, output formats, exit codes).

  5. No testing strategy. How do we test reconciliation quality? Integration tests with a real LLM vs mocked? Benchmark suite for recall quality?
